## Requirements
### Requirement: Brief-keyed command entry point
The plugin SHALL provide a thin command `/ssc.image-prompt <brief_id>` that holds no
orchestration logic and dispatches `ssc-image-prompt-agent`. `brief_id` SHALL be the
sole required input; the owning idea SHALL be resolved from the brief via `get_brief`
(never passed in). The command SHALL accept optional `step`/`stage`, `revise:`,
`change:` (Edit), and `product:` arguments and pass them through unchanged.

#### Scenario: Dispatch with a brief id
- **WHEN** an operator runs `/ssc.image-prompt <brief_id>`
- **THEN** the command dispatches `ssc-image-prompt-agent` with that `brief_id`, resolving the idea from the brief, and performs no content work itself

#### Scenario: Missing brief id
- **WHEN** an operator runs `/ssc.image-prompt` with no `brief_id`
- **THEN** the command asks the operator for the `brief_id` and invents nothing

### Requirement: Precondition gating
The agent SHALL resolve `get_brief(brief_id) → { brief, idea }` and gate before doing
any work: the idea MUST be `channel='ad'` and `status='approved'`, and the brief MUST
be `status='approved'`. On any failed gate the agent SHALL STOP with a Vietnamese
message routing the operator to the correct prior step and SHALL write nothing.

#### Scenario: Non-ad idea
- **WHEN** the resolved idea has `channel !== 'ad'`
- **THEN** the agent STOPs (Vietnamese) explaining the prompt flow currently runs only for ad concepts, and writes nothing

#### Scenario: Unapproved brief
- **WHEN** the resolved brief has `status !== 'approved'`
- **THEN** the agent STOPs (Vietnamese) asking the operator to approve an angle brief first, and writes nothing

### Requirement: Five-step optional chain with a next-open-step state machine
The agent SHALL be a state-driven stepper over the ordered steps
`[scene, subject, composition, edit, text]`, **all of which are OPTIONAL**. On each
invocation it SHALL resolve studio state from `list_creatives`,
`list_creative_prompts`, `list_content`, and `list_gallery_media`, work the **single
next-open step** by dispatching that step's skill, and STOP. It SHALL NOT advance more
than one step per invocation, SHALL NOT fan out, and SHALL NOT generate.

#### Scenario: Author the next-open step
- **WHEN** the next-open step has no saved prompt yet
- **THEN** the agent dispatches that step's skill, which authors the prompt + `generation_config`, saves it via `save_creative_prompt`, and STOPs telling the operator to Generate in the studio

#### Scenario: Wait at a human gate
- **WHEN** the next-open step already has a saved prompt but no generated/selected candidate, and no `revise:` was given
- **THEN** the agent STOPs asking the operator to Generate + select a candidate in the ImageStudio, and writes nothing

#### Scenario: Steps are skippable
- **WHEN** an operator targets a later step directly and the earlier steps have no selected candidate
- **THEN** the pipeline still proceeds provided that step's own preconditions are met, because every step is optional and skip-transparent

### Requirement: Skill-to-layer mapping
Each stage-skill SHALL persist exactly its own layer: `ssc-image-prompt-scene` →
`layer:'scene'`; `ssc-image-prompt-subject` → `layer:'subject'`;
`ssc-image-prompt-composition` → `layer:'composition'`; `ssc-image-prompt-edit` →
`layer:'edit'`; `ssc-image-prompt-text` → `layer:'text'`. No skill SHALL save a prompt
with `layer:'product'` — the product is an upload-only input, never a prompt step.

#### Scenario: Each skill saves its own layer
- **WHEN** any stage-skill persists its prompt
- **THEN** the `layer` argument equals that skill's own layer key, and the skill name, the studio label, and the layer key all agree

#### Scenario: Product is never a prompt layer
- **WHEN** any step runs
- **THEN** no step calls `save_creative_prompt` with `layer:'product'`

### Requirement: Scene is text-to-image only and takes no real references
The Scene step SHALL author a complete, filled text-to-image image and MAY freely
depict a **generic** subject and/or product, or neither, as the brief needs. It SHALL
NOT take any real reference or anchor (no identity model, no real-model photo, no
product-packshot pool id, no `controlSourceRef`), and it SHALL NOT fabricate the real
branded Cambridge packaging — any depicted product MUST be generic and unbranded.

#### Scenario: Generic subject permitted
- **WHEN** the brief calls for a person in the frame
- **THEN** Scene describes a generic, illustrative persona-matched person in words, generated by the text-to-image model, with no identity reference

#### Scenario: Real packshot deferred
- **WHEN** the brief involves the Cambridge product
- **THEN** Scene keeps any depicted product generic and unbranded, and the real packshot is brought in later by Composition or Edit as a reference

### Requirement: No reserved planes
No step SHALL carve out a reserved text zone or a reserved subject zone. The Scene
SHALL be authored as a complete, filled image. Text headroom MAY be expressed only as
an optional positive **framing** choice (composing so a naturally quieter area falls
where a headline may later sit) and SHALL NEVER be a reserved empty band, a described
void, or a "leave room for the headline" instruction.

#### Scenario: Complete scene, not an empty plate
- **WHEN** the Scene step authors its prompt
- **THEN** it describes a full, lived-in environment and describes no empty bands, cleared planes, or reserved voids

#### Scenario: Text needs no cleared plane
- **WHEN** the Text step later renders the headline
- **THEN** it places text onto a naturally clean area of the finished image, requiring no pre-cleared plane from any upstream step

### Requirement: Subject is a standalone person with face and pose locked
The Subject step SHALL author a prompt for the persona's person generated ALONE on a
simple ground, and SHALL lock identity at this step: when a real-model photo is
resolvable in the pool it MUST select an identity model and set `identityRef` plus
`idWeight` (optionally `controlType:'pose'` with `controlSourceRef` and
`conditioningScales` as an operator opt-in); otherwise it MUST select a text-to-image
model, describe a persona-matched person, and leave `identityRef` unset while saying so.

#### Scenario: Face locked with an identity reference
- **WHEN** a real-model photo is resolvable in the gallery pool
- **THEN** the Subject step sets an identity model with `identityRef` and `idWeight`

#### Scenario: No identity reference resolvable
- **WHEN** no identity photo is resolvable
- **THEN** the Subject step selects a text-to-image model, describes the person from the persona doc, leaves `identityRef` unset, and tells the operator they may attach a reference in the studio

### Requirement: Composition is anchor-gated
The Composition step SHALL require at least ONE anchor — a selected `subject` OR an
approved brief `product`. A selected Scene ALONE SHALL NOT satisfy the gate. With
neither anchor the step SHALL STOP and author nothing. With a selected Scene it SHALL
author an edit that composes the anchor(s) onto that Scene (the Scene being the edit
base); without a Scene it SHALL author an image built around the anchor(s).

#### Scenario: Neither anchor present
- **WHEN** the Composition step runs with no selected subject and no approved product
- **THEN** it STOPs (Vietnamese), authors nothing, and explains that a Scene alone is not an anchor

#### Scenario: Anchors composed onto a Scene
- **WHEN** a selected Scene and at least one anchor exist
- **THEN** the step authors a reference-edit composing the anchor(s) onto that Scene as the edit base

### Requirement: Edit is a generic, optional, repeatable change over the chain tip
The Edit step SHALL author ONE generic reference-edit describing the operator's
requested change (`change:`) applied to the chain tip. It SHALL be optional and
repeatable, chaining edit-on-edit. Product placement SHALL be treated as one kind of
edit, not a mandatory dedicated step; when the change touches the product, the
real-packaging fidelity discipline MUST apply and a product MUST NEVER be fabricated.

#### Scenario: A requested change is authored
- **WHEN** the operator supplies a `change:` instruction and a chain tip exists
- **THEN** the Edit step authors one reference-edit body for that change, saves `layer:'edit'`, and STOPs

#### Scenario: No change supplied
- **WHEN** the Edit step runs with no `change:`, no `revise:`, and no pending saved edit
- **THEN** it STOPs and asks the operator what to change, authoring nothing

### Requirement: Chain-tip lineage resolution
Steps that build on a prior image SHALL resolve the chain tip as the **nearest previous
selection**, walking `['edit', 'composition', 'subject', 'scene']` nearest-first, with
skipped optional steps transparent and `discarded` rows ignored. The Text step SHALL
NOT be anchor-gated — a Scene alone is a valid parent.

#### Scenario: Nearest previous selection wins
- **WHEN** a brief has a selected Scene and a selected Composition but no Edit
- **THEN** the chain tip resolves to the Composition

#### Scenario: Text on a Scene alone
- **WHEN** only a Scene has been selected
- **THEN** the Text step may still parent on it, because Text is not anchor-gated

### Requirement: Text renders the exact approved headline
The Text step SHALL require a chain tip and an approved `image_content` row, and SHALL
author a placement prompt carrying the EXACT approved Vietnamese string verbatim,
selecting the legible in-image text model or the deterministic diacritic-safe overlay.
With no approved `image_content` it SHALL STOP and route the operator to
`/ssc.ads-produce <brief_id> image_content`.

#### Scenario: Exact headline carried
- **WHEN** the Text step runs with an approved `image_content` row
- **THEN** its prompt carries the verbatim Vietnamese string as the text-render input

#### Scenario: Overlay preferred for diacritics
- **WHEN** correct Vietnamese diacritics must be guaranteed
- **THEN** the step selects the deterministic overlay pseudo-model instead of the in-image text model

### Requirement: Full generation_config authoring
Each step SHALL author the prompt `body` PLUS a `generation_config` carrying the chosen
model and only the capability inputs that model's profile declares (`controlType`,
`controlSourceRef`, `identityRef`, `conditioningScales`, `idWeight`). Text-to-image
steps MUST use the FLUX.2 ladder; reference-edit steps MUST use a Kontext-family or
control-capable model. A reference id SHALL be named only when resolvable, never guessed.

#### Scenario: Capability-matched settings
- **WHEN** a step selects a plain text-to-image model
- **THEN** its `generation_config` sets `model` only and omits control/identity fields

#### Scenario: Unresolvable reference
- **WHEN** an identity or control reference cannot be resolved from the pool
- **THEN** the step saves without that reference and tells the operator to attach it in the studio

### Requirement: Grounding in all approved brief contents
Every step SHALL ground its prompt in ALL approved contents of the brief — approved
`copy`, `headline`, `description`, and `image_content` read via `list_content` — using
them for MEANING and TONE only. Except for the Text step's exact string, no step SHALL
name any of those strings in a prompt.

#### Scenario: Meaning without words
- **WHEN** an upstream step grounds its prompt in approved copy
- **THEN** it describes the scene that copy implies and names none of its words

### Requirement: Selected-image prompts are read from provenance
A step that grounds on a previously selected image SHALL read that image's prompt from
`list_creatives → creatives[].media.provenance.prompt` (the immutable per-image record).
The `creatives` row carries no `generation_prompt` column;
`list_creative_prompts` is only a secondary fallback for the authored recipe.

#### Scenario: Grounding on a selected parent
- **WHEN** a step needs the light, perspective, or palette of a selected upstream image
- **THEN** it reads that image's `media.provenance.prompt` rather than any creative-row prompt field

### Requirement: Propose-only, zero-credit governance
The agent and every stage-skill SHALL hold only read tools plus `save_creative_prompt`.
They SHALL NOT hold or call any generate tool (`generate_scene`, `generate_subject`,
`generate_composition`, `edit_creative`, `generate_text_layer`), `approve` /
`unapprove`, `upload_creative` / `confirm_creative_upload` / `select_gallery_creative`,
`set_cover`, `reorder_gallery`, any publish tool, or `update_budget`. Saving a prompt
SHALL NOT approve anything and SHALL spend no credits.

#### Scenario: No generation tool present
- **WHEN** the agent or a stage-skill frontmatter `tools:` list is read
- **THEN** it contains reads + `save_creative_prompt` only

#### Scenario: Generation stays human
- **WHEN** a step's prompt is saved
- **THEN** no image is generated and no candidate is selected by the agent — both remain human ImageStudio actions

### Requirement: Prompt discipline
Every authored prompt SHALL describe only the desired end-state positively. No step
SHALL negate, and no step except Text SHALL name any approved-content string in any
form (quoted, paraphrased, or negated). No step SHALL bake text into an image, and no
step SHALL describe reserved voids.

#### Scenario: Positive phrasing
- **WHEN** a step wants an uncluttered surface
- **THEN** it describes what is positively there rather than negating what is absent

#### Scenario: Text is the sole naming exception
- **WHEN** the Text step authors its prompt
- **THEN** it names the exact approved string, and no other step names any approved-content string

### Requirement: Revise handling
When `revise: <note>` is supplied, the agent SHALL apply it to the active step's prompt
— rewriting the saved prompt with `expected_version` for optimistic concurrency and
re-saving — and SHALL never drop the note, never change which step is active, and never
generate.

#### Scenario: Revise the active step
- **WHEN** `revise:` is supplied and the active step has a saved prompt
- **THEN** the agent rewrites that step's prompt, saves it with `expected_version`, and STOPs without generating

### Requirement: Upstream staleness warning
The agent or skill SHALL surface a Vietnamese warning — and SHALL NEVER block the work —
when a step that already has a selected candidate is reopened or revised while a later
step also has a selected candidate. The warning MUST state that editing the prompt does
not change the already-selected image and that reselecting a different image would leave
the downstream steps stale.

#### Scenario: Warned, then proceeds
- **WHEN** an operator revises a step whose downstream steps already have selected candidates
- **THEN** the warning is surfaced and the revision still proceeds

### Requirement: Coexistence with the existing ssc.image command
This pipeline SHALL be added alongside the existing `/ssc.image` command and SHALL NOT
remove it. The command's frontmatter description SHALL state that it is the
propose-only, zero-credit sibling so operators can pick deliberately.

#### Scenario: Existing command untouched
- **WHEN** this change is applied
- **THEN** `/ssc.image` remains available and the new command declares itself the propose-only sibling

### Requirement: Deployment-dependency safe stop
The skill SHALL STOP cleanly with a Vietnamese message and write nothing when the
deployed BrandOS server rejects a `save_creative_prompt` layer or config it relies on,
asking an admin to deploy the newer server, and SHALL NOT retry in a loop.

#### Scenario: Layer not supported
- **WHEN** `save_creative_prompt` is rejected because the deployed server does not support the layer
- **THEN** the skill STOPs (Vietnamese), writes nothing, and does not retry

### Requirement: Vietnamese operator-facing prose
All operator-facing chat and persisted Vietnamese notes SHALL be Vietnamese. Image
prompt `body` values are free-form EXCEPT the Text step's string, which SHALL be the
exact Vietnamese approved content.

#### Scenario: Vietnamese stop messages
- **WHEN** any skill or the agent STOPs and addresses the operator
- **THEN** the message is in Vietnamese
