## User Input

```text
$ARGUMENTS
```

Consider the user input above before proceeding (if not empty). Expected inputs — **one of**:

- **Ad concept idea ID** (`idea_id`) — the id of ONE approved ad concept (an `ideas` row, `channel='ad'`, `status='approved'`). This is the key the dispatched skill reads and writes against.
- **Date** (`date`, format `YYYY-MM-DD` — a calendar day, e.g. `2026-07-14`). Resolved to the approved ad concept(s) for that day. If several concepts are scheduled that day, the dispatched skill works ONE concept per invocation.

If neither `idea_id` nor `date` is given, ask the operator for one (one question) before dispatching. Do not invent one. There is **no** `section` and **no** `brief_id` here — this command only **generates** the angle set; producing text from a chosen angle is `/ssc.ads-produce`.

Optional:

- **Hero revision** (`revise hero: <note>`) — a trailing free-text instruction, passed through to the dispatched skill verbatim. Use it **only** to change a concept's already-defined **hero**: the single Vietnamese north-star sentence naming the one concrete product/feature/pain-point the concept is about, which `ssc-ads-brief` defines once (before any angle exists) and every angle + every copy variation downstream must stay faithful to. Omit it and the hero is resolved automatically — defined from the concept `title` when the idea has none, left untouched when it already has one. A revision never edits, re-scores, or blocks the briefs and copy already written under the old hero; the skill just names which existing briefs predate it.
- **Period** (`period`, format `YYYY-MM`) — informational only; the month the concept belongs to, used when pointing the operator at `/ad/[month]/[id]`. The dispatched skill resolves everything from the `idea_id`/`date`.

This command is the **first step** of the brief-first Ads production flow. It runs **after** the Ads pipeline's **Ideate** step — a concept is only worked once it has been ideated and **approved** in the dashboard — and **before** any copy. It reads **no** `channel_plan` gate flags and requires **no** approved copy. There is **no** `/ssc.ads-plan` precondition beyond an approved concept.

## What to do

This command is a thin entry point — it holds **no** orchestration logic. It dispatches **`ssc-ads-brief`** (`idea_id` [, `date`] [, a `revise hero: <note>` instruction passed through verbatim]) and stops.

`ssc-ads-brief` is an **append-and-curate angle generator that runs first**: with **no copy precondition**, it judges which personas — from the live persona roster — the persona-free concept genuinely fits, then per fitting persona derives **distinct, rated DRAFT creative-brief angle briefs** from the concept (`title`) + that persona's own detail doc — each anchored to a *different* trigger point / objection / myth in her doc, each carrying a mandatory short Vietnamese `angle_label` naming both the persona and the anchor, the five narrative fields (`hook_direction`/`core_message`/`why_now`/`story_moment`/`cta` — never `theme`, which ad ideas don't carry), its declared media home (`awareness_stage` + the tier it implies), and a 1–5 score + Vietnamese comment. It self-scores each angle, drops + regenerates any rated ≤3 until every saved angle is ≥4, then **saves each as a DRAFT `brief`** via `save_brief` (which mints DRAFT briefs only) and **stops**. Briefs live in a `briefs` table (not on the idea row, not a `content` row) — **each angle is its own brief row with its own `angle_label` and `brief_id`**, so an idea genuinely carries several angle briefs, across several personas. The operator reviews and **approves the angle(s) worth producing**; each approved angle then anchors its own independent production run.

### What a bare `/ssc.ads-brief <idea_id>` does — every time

**It appends new angles and curates its own DRAFT set.**

- **On a concept with no briefs yet (cold start)** — you get **up to five** distinct rated angles, saved as DRAFT briefs.
- **On a concept that ALREADY has briefs** — it does **not** stop. It reads **all** of them (**any status** — approved *and* draft alike) as the **taken set**, compares against their five narrative fields (not just their labels), and **appends only the angles that genuinely remain** — however many that honestly is. It then **curates the DRAFT rows**: a weak, mis-homed, hero-drifted, or near-duplicate draft is revised in place (re-scored) or, when it is a redundant **no-cost** draft (0 creatives AND 0 copy), discarded via a preview-then-confirm cascade. **APPROVED briefs are left exactly as they were** — never edited, never deleted; a draft carrying produced work it cannot discard is left in place and reported to you. A healthy draft set is left untouched (curation is not churn).
- **When no distinct new angle remains** — it appends **nothing** and says so plainly, in Vietnamese, naming the reason (the concept's distinct angles are exhausted) and suggesting you sharpen the concept (`title`) if you genuinely want more. **An empty append is an ordinary, successful outcome** (curation may still have run) — the skill never pads the set with a near-duplicate angle.

**So: want another angle? Just re-run this command** — it appends whatever genuinely remains and tidies the drafts. Approved angles are never touched.

## Governance

Nothing auto-approves, auto-applies, or auto-publishes. `ssc-ads-brief` **saves DRAFTS** and stops: NEW DRAFT `brief` rows (one row per rated angle) via `save_brief`, **appended alongside** whatever the concept already carries. The operator reviews / curates and **approves the angle(s) worth producing** on the `/ad/[month]/[id]` page. **"Save" persists drafts; it is NOT approval** — it never flips a gate and never touches an idea's `status`. Propose-only in the approval sense (hard rule): the skill never changes approval or lifecycle state in either direction — never `approve` (the ONLY gated promotion, denied to agents by the approval hook; any entity incl. `brief`, any gate), never un-approves, never publishes. **It DOES curate its own DRAFT briefs:** `save_brief` appends new drafts, and `edit`/`delete` (generic verbs, used only with `entity='brief'` on this concept's DRAFT rows) revise or discard them. **APPROVED briefs are the operator's — never edited or deleted by the skill.** The server enforces the hard edges (un-approving and deleting a brief with produced work both need the `approve` capability the skill lacks); the approved-brief-immutability of a plain field edit is a skill rule on top. Its one idea-row write is `update_idea(hero=…)`, a single idea-wide field written at most once per run (when the concept has no hero yet, or the operator asked to revise it) — it touches no brief, no status, and no gate. All persisted prose (the brief angle fields, labels, and comments) is **Vietnamese**. Producing requires `edit` (plus `view` for the resolve reads); approving a draft brief later requires `approve` on the page.

**Discarding an angle is a gated, preview-then-confirm CASCADE.** Deleting a brief hard-removes it together with every creative link, every bound copy row, and every prompt — **no tombstone**, the angle is gone for good — and previews the blast radius before it commits. Who may run it is scoped by what the angle carries: the **skill itself discards only a no-cost DRAFT** (0 creatives AND 0 copy) as part of curation; anything with **produced work needs the `approve` capability**, so a brief with creatives/copy — and any **approved** brief (un-approve it first) — is **yours to discard in the dashboard**, deliberately, with the blast radius in view. When the skill meets a draft it would tidy away but cannot (produced work / approved), it leaves it and tells you. Discarding is never how you get *new* angles — re-running this command is.

## After it runs

After `ssc-ads-brief` appends the new angle(s), point the operator to `/ad/[month]/[id]` for the concept to **review them and approve the one(s) worth producing**. An angle you don't want is simply an angle you never approve — it costs nothing to leave it sitting as a draft. Each approved angle is an **independent production track**, anchored to its own `brief_id`: produce that angle's ad text with **`/ssc.ads-produce <brief_id>`** (copy first, then `headline`/`description`/`image_content`), and its visual with **`/ssc.image-prompt <brief_id>`** — one `brief_id` per run, and the approved angles can be produced in any order or left unproduced.

**Want more angles later? Re-run `/ssc.ads-brief <idea_id>`.** It appends whatever distinct angles still remain — including a sixth angle on a concept that already carries five approved briefs — and tidies the DRAFT rows (revise weak, discard no-cost redundant), **without touching any approved angle or the copy/creatives anchored to it**. If nothing distinct remains to append, it appends nothing and tells you so (see "What a bare `/ssc.ads-brief <idea_id>` does"). If you genuinely want more angles, sharpen the concept (`title`) and re-run — discarding a draft is curation, never a way to unlock more angles.

Re-invoke per concept — it works ONE approved concept at a time.
